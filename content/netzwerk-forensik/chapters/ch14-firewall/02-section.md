<h2 class="section-title"><span class="number">14.2</span> iptables &ndash; Das Standard-Tool</h2><p><strong>iptables</strong> ist das klassische Firewall-Tool unter Linux. Es arbeitet mit <strong>Ketten</strong> (Chains) und <strong>Regeln</strong> (Rules).</p><p><strong>Die drei Hauptketten:</strong></p><div class="table-container"><table><thead><tr><th>Kette</th><th>Richtung</th><th>Erkl&auml;rung</th><th>Beispiel</th></tr></thead><tbody><tr><td><strong>INPUT</strong></td><td>Eingehend</td><td>Verbindungen, die an dein Ger&auml;t gerichtet sind</td><td>Web-Server auf Port 80</td></tr><tr><td><strong>OUTPUT</strong></td><td>Ausgehend</td><td>Verbindungen, die von deinem Ger&auml;t starten</td><td>Browser greift auf Webseite zu</td></tr><tr><td><strong>FORWARD</strong></td><td>Weitergeleitet</td><td>Verbindungen, die durch deinen Rechner geroutet werden</td><td>Linux als Router</td></tr></tbody></table></div><div class="code-block"><div class="code-header"><span class="lang">BASH</span><button class="copy-btn">Kopieren</button></div><pre><code># Alle Regeln anzeigen (detailiert)
sudo iptables -L -n -v

# SSH erlauben (Port 22) - WICHTIG: Erst SSH erlauben!
sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# HTTP erlauben (Port 80)
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT

# HTTPS erlauben (Port 443)
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# Alles andere blockieren (als LETZTE Regel!)
sudo iptables -A INPUT -j DROP</code></pre></div><div class="table-container"><table><thead><tr><th>Parameter</th><th>Bedeutung</th><th>Beispiel</th></tr></thead><tbody><tr><td><span class="inline-code">-A</span></td><td>Regel am Ende anh&auml;ngen</td><td><span class="inline-code">-A INPUT</span></td></tr><tr><td><span class="inline-code">-I</span></td><td>Regel vorne einf&uuml;gen</td><td><span class="inline-code">-I INPUT 1</span> (an Position 1)</td></tr><tr><td><span class="inline-code">-D</span></td><td>Regel l&ouml;schen</td><td><span class="inline-code">-D INPUT 1</span></td></tr><tr><td><span class="inline-code">-p</span></td><td>Protokoll</td><td><span class="inline-code">-p tcp</span> oder <span class="inline-code">-p udp</span></td></tr><tr><td><span class="inline-code">--dport</span></td><td>Ziel-Port</td><td><span class="inline-code">--dport 80</span></td></tr><tr><td><span class="inline-code">-s</span></td><td>Quell-IP</td><td><span class="inline-code">-s 192.168.1.0/24</span></td></tr><tr><td><span class="inline-code">-j</span></td><td>Aktion</td><td><span class="inline-code">ACCEPT</span>, <span class="inline-code">DROP</span>, <span class="inline-code">REJECT</span></td></tr></tbody></table></div><p><strong>ACCEPT vs DROP vs REJECT:</strong></p><ul><li><span class="inline-code">ACCEPT</span> = Verbindung erlauben</li><li><span class="inline-code">DROP</span> = Verbindung stillschweigend ignorieren (keine Antwort)</li><li><span class="inline-code">REJECT</span> = Verbindung ablehnen mit Fehlermeldung</li></ul><div class="callout callout-warning"><div class="callout-header">&#9888; Achtung &ndash; Nicht aussperren!</div><p>Regeln werden <strong>von oben nach unten</strong> abgearbeitet. Die erste passende Regel gewinnt. Wenn du <span class="inline-code">DROP ALL</span> vor <span class="inline-code">ACCEPT SSH</span> setzt, bist du ausgesperrt! <strong>Immer zuerst ACCEPT f&uuml;r SSH, dann DROP.</strong></p></div>
