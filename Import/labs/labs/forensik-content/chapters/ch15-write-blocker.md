<h1 class="chapter-title">Write-Blocker</h1><div class="chapter-subtitle">Hardware & Software - Original vor Ver&auml;nderungen sch&uuml;tzen</div><div class="callout callout-context"><div class="callout-header">&#9432; Warum ist das forensisch wichtig?</div><p>Ein Write-Blocker verhindert, dass das Untersuchungssystem auf den Original-Datentr&auml;ger schreibt. Ohne Write-Blocker kann versehentlich oder absichtlich der Datentr&auml;ger ver&auml;ndert werden, was die forensische Integrit&auml;t zerst&ouml;rt.</p></div>

<p class="chapter-intro">Write-Blocker verhindern, dass das Original während der Untersuchung verändert wird. Sie sind der wichtigste Schutz für Beweismittel.</p>

<div class="callout callout-info"><div class="callout-header">&#9432; Was du nach diesem Kapitel kannst</div><p>Du kennst die Funktionsweise von Write-Blockern (Hardware und Software), verstehst warum sie essenziell sind und weisst wann du sie einsetzt.</p></div>

<h2 class="section-title"><span class="number">13.1</span> Hardware-Write-Blocker</h2><p>Physische Hardware-Ger&auml;te, die Writes blocken.</p><div class="code-block"><div class="code-header"><span class="lang">BASH</span><button class="copy-btn">Kopieren</button></div><pre><code># Hardware-Write-Blocker verifizieren
lsblk -o NAME,RO

# RO=1 bedeutet Read-Only (Write-Blocker aktiv)
lsblk -o NAME,RO,SIZE,TYPE,MOUNTPOINT,MODEL</code></pre></div><div class="table-container"><table><thead><tr><th>Typ</th><th>Vorteile</th><th>Nachteile</th></tr></thead><tbody><tr><td>USB Write-Blocker</td><td>Billig, universell kompatibel</td><td>Kann umgangen werden, fragil</td></tr><tr><td>PCIe Write-Blocker</td><td>Schnell, zuverl&auml;ssig, schwer zu umgehen</td><td>Teuer, ben&ouml;tigt freien PCI-Slot</td></tr><tr><td>eSATA/SATA Write-Blocker</td><td>Kompatibel mit SATA-Devices</td><td>L&auml;ngsamer als PCIe, kann umgangen werden</td></tr></tbody></table></div><div class="callout callout-warning"><div class="callout-header">&#9888; Wichtig</div><p>Hardware-Write-Blocker k&ouml;nnen von kompromittierten Systemen umgangen werden. Verifiziere den RO-Status vor dem Imaging!</p></div><h2 class="section-title"><span class="number">13.2</span> Software-Write-Blocker</h2><p>Kernel-Level Protection, die Writes blockt.</p><div class="code-block"><div class="code-header"><span class="lang">BASH</span><button class="copy-btn">Kopieren</button></div><pre><code># Linux Kernel Write-Blocker (SysRq)
echo 1 > /proc/sys/kernel/sysrq
echo w > /proc/sysrq-trigger

# Pr&uuml;fen ob RO gesetzt ist
lsblk -o NAME,RO

# Reset (nur f&uuml;r Entwicklungs-Systeme!)
echo 0 > /proc/sys/kernel/sysrq</code></pre></div><div class="table-container"><table><thead><tr><th>Option</th><th>Beschreibung</th></tr></thead><tbody><tr><td>1</td><td>Alle SysRq-Funktionen aktivieren</td></tr><tr><td>w</td><td>Alle Schreibzugriffe blocken (Write-Blocker)</td></tr><tr><td>0</td><td>SysRq deaktivieren</td></tr></tbody></table></div><div class="callout callout-danger"><div class="callout-header">&#9888;
