# Build System

## Overview

`build_content.py` is the build script that reads Markdown content files and generates a single JavaScript file (`content-data.js`) containing all chapter HTML.

## Usage

```bash
python3 build_content.py
```

### Prerequisites

- Python 3.6+
- PyYAML (`pip install pyyaml`)

## Input Structure

```
content/
  <lab-id>/
    meta.yaml                     # Required: lab metadata
    chapters/
      <chapter-id>/
        00-intro.md               # Required: first file must have YAML frontmatter
        01-section.md             # Optional: additional sections
        02-section.md
```

### `meta.yaml` Format

```yaml
id: "linux-forensik"              # Required: unique lab identifier
title: "Linux Forensik Lab"       # Required: display title
icon: "&#128270;"                 # Required: HTML entity for icon
description: "..."                # Required: short description for hub card
accent: "#00d4aa"                 # Optional: CSS accent color (default: #00d4aa)
canonical_order:                  # Required: chapter ordering
  - welcome
  - ch01-grundlagen
  - ch02-identifikation
```

### Chapter Markdown Format

First `.md` file in each chapter directory must have YAML frontmatter:

```markdown
---
id: "ch01-grundlagen"
title: "Grundlagen"
icon: "&#9881;"
section: "Trainingsmodul 1"
prev: "welcome"
next: "ch02-identifikation"
---

<h1 class="chapter-title">Grundlagen</h1>
<p>Content here...</p>
```

Subsequent `.md` files are pure HTML (no frontmatter):

```markdown
<h2 class="section-title">Devices</h2>
<p>Content here...</p>
```

### File Naming Convention

All `.md` files must use numeric prefixes for deterministic ordering:

```
00-intro.md       # Parsed first, must contain frontmatter
01-section.md     # Second section
02-section.md     # Third section
```

### HTML Preservation

The build script preserves HTML content exactly as-is. No markdown-to-HTML conversion occurs. This ensures zero visual regression.

## Output Format

Generates `forensik-lab/assets/js/content-data.js`:

```javascript
// Auto-generated by build_content.py - DO NOT EDIT MANUALLY

var LabRegistry = [
  { id: "linux-forensik", title: "Linux Forensik Lab", icon: "...", description: "...", accent: "#00d4aa" },
];

var ContentData = {};

ContentData["linux-forensik"] = {};

ContentData["linux-forensik"]["welcome"] = `<html>...`;
ContentData["linux-forensik"]["ch01-grundlagen"] = `<html>...`;

var ContentNav = {};

ContentNav["linux-forensik"] = [
  { id: "welcome", label: "Willkommen", icon: "...", section: "Start" },
  { id: "ch01-grundlagen", label: "Grundlagen", icon: "...", section: "Trainingsmodul 1" },
];
```

### Template Literal Escaping

Content is wrapped in JavaScript template literals (backticks). The following characters are escaped:

- `` ` `` → `` \` ``
- `${` → `\${`
- `\` → `\\`

## Lab Discovery

Labs are auto-discovered by scanning `content/*/meta.yaml`. No registration or configuration needed — just create a new directory with a `meta.yaml` and chapters.

## Validation

The build script validates:

1. `meta.yaml` has all required fields (`id`, `title`, `icon`, `description`)
2. Each chapter's first `.md` file has YAML frontmatter with (`id`, `title`, `icon`, `section`)
3. No empty required fields
4. All filenames have numeric prefixes

Validation errors halt the build with clear error messages.

## Adding a New Lab

```bash
# 1. Create lab directory
mkdir -p content/netzwerk-forensik/chapters/welcome

# 2. Create meta.yaml
cat > content/netzwerk-forensik/meta.yaml << 'EOF'
id: "netzwerk-forensik"
title: "Netzwerk Forensik Lab"
icon: "&#127760;"
description: "Netzwerkforensische Analyse"
accent: "#3b82f6"
canonical_order:
  - welcome
EOF

# 3. Create first chapter
cat > content/netzwerk-forensik/chapters/welcome/00-intro.md << 'EOF'
---
id: "welcome"
title: "Willkommen"
icon: "&#127760;"
section: "Start"
---
<h1 class="chapter-title">Netzwerk Forensik</h1>
EOF

# 4. Build
python3 build_content.py
```
